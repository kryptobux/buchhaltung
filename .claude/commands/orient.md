description: Lagebild aus STATE + KANBAN (Sessionstart)
---

Lies STATE.md und KANBAN.md (nur die Abschnitte 📥 Backlog und 🔨 In Arbeit) sowie `git log -3 --oneline` und `git status -s`.

Gib dann ein Lagebild in GENAU 3 Zeilen:
1. **Stand:** Prod-/main-Stand + letzter Commit.
2. **Läuft:** aktueller Task bzw. „nichts in Arbeit" + nächster Schritt laut STATE.
3. **Achtung:** offene Operator-Aktionen / fremde ungecommittete Änderungen / Abweichungen zwischen STATE und Repo-Realität.

Keine Rohinhalte der Dateien wiedergeben. Danach auf Anweisung warten — außer der User hat schon eine Aufgabe genannt.
